MY DESIGN PROCESS

Think

Identify and understand the problem. This is the first and most important step in my design process. Once I know what the problem is and can understand it I progress to the next stage.

Draw

Using pen and paper (or whiteboards!) I through down some UI ideas to solve the problem users are having. Of course, this only happens when a UI is part of the solution. Many times it isn’t. Feedback and discussions abound.

Code

emacs > vim

After getting feedback and iterating on designs, it’s time to write some code. Using throwaway HTML/CSS code, I replicate the final product as closely as possible before getting more feedback. Once the solution is close to what we want, developers write the ‘real’ code and boom, it gets shipped. Visual polish also happens at this stage.
